# 虚拟试穿系统 — 部署与启动指南

## 一、环境要求

| 依赖 | 版本 | 说明 |
|------|------|------|
| Node.js | >= 18 | 前端运行 |
| Python | >= 3.11 | 后端运行 |
| Docker | >= 24 | PostgreSQL + Redis |
| npm | >= 9 | 前端包管理 |
| pip | >= 23 | 后端包管理 |

## 二、快速启动（开发环境）

### 1. 克隆项目

```bash
git clone <repo-url>
cd chuanda
```

### 2. 启动数据库

```bash
docker-compose up -d
```

启动 PostgreSQL (端口 5432) 和 Redis (端口 6379)。

### 3. 配置环境变量

```bash
# 后端环境变量
cp backend/.env.example backend/.env
# 编辑 backend/.env，填入实际的 API Key
```

需要配置的 Key：
- `OPENAI_API_KEY` — 小米 mimo 中转站 API Key
- `OPENAI_IMAGE_API_KEY` — hfsyapi 中转站 API Key
- `JWT_SECRET` — JWT 密钥（生产环境必须修改）

### 4. 启动后端（Conda 环境）

```bash
cd backend

# 创建 conda 环境（首次）
conda env create -f environment.yml

# 激活环境
conda activate tryon-backend

# 初始化管理员账号
python init_admin.py

# 启动服务
uvicorn app.main:app --reload --port 8000
```

如果不使用 conda，也可以直接 pip：
```bash
pip install -r requirements.txt
```

后端启动后访问：
- API 文档：http://localhost:8000/docs
- 健康检查：http://localhost:8000/health

### 5. 启动前端

```bash
# 回到项目根目录
cd ..

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端访问：http://localhost:3000

### 6. 访问后台管理

打开 http://localhost:3000/admin/login

默认管理员账号：
- 用户名：`admin`
- 密码：`admin123`

### 7. 创建测试用户（可选）

```bash
# 在 Windows 终端执行
node prisma/seed-user.js
```

创建测试用户：chenchenclaw@gmail.com / chenchen123

## 三、生产环境部署

### 1. Docker Compose 一键部署

```bash
# 修改 backend/.env 中的密钥和 API Key
# 修改 docker-compose.yml 中的端口映射

docker-compose up -d --build
```

### 2. 前端构建

```bash
npm run build
npm start
```

### 3. Nginx 反向代理（参考）

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端
    location / {
        proxy_pass http://localhost:3000;
    }

    # 后端 API
    location /api/ {
        proxy_pass http://localhost:8000;
    }

    # 上传文件
    location /uploads/ {
        proxy_pass http://localhost:8000;
    }
}
```

## 四、常用命令

```bash
# 查看后端日志
docker-compose logs -f backend

# 进入 PostgreSQL
docker exec -it tryon-postgres psql -U tryon -d virtual_tryon

# 重启后端
docker-compose restart backend

# 停止所有服务
docker-compose down

# 清除数据（谨慎）
docker-compose down -v
```

## 五、故障排查

| 问题 | 解决方案 |
|------|---------|
| 数据库连接失败 | 检查 Docker 是否启动：`docker ps` |
| 后端启动报模块缺失 | `pip install -r requirements.txt` |
| 前端 API 调用 404 | 检查 `NEXT_PUBLIC_API_URL` 是否正确 |
