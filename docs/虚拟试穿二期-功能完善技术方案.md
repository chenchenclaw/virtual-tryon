## 二、P0：多件搭配分步试穿

### 2.1 问题分析

当前 MVP 的试穿逻辑：将用户照片 + 所有服装图片一次性传入 `reference_images`（最多4张），由 AI 一次性生成最终效果图。

**问题**：
- 超过 3 件服装时无法全部传入 reference_images
- 一次性生成多件服装的效果质量不如逐层叠加
- 无法展示中间过程（如只穿上衣的效果）

### 2.2 技术方案：分步叠加生成

```
第一步：用户照片 + 上衣图片 → 生成穿上衣的效果图
第二步：上衣效果图 + 裤装图片 → 生成完整上身+下身效果图
第三步（可选）：完整效果图 + 鞋履图片 → 生成带鞋的最终效果图
```

每步将上一步的结果作为新的 reference_images[0]，实现服装逐步叠加。

### 2.3 核心改动

#### 2.3.1 tryon.service.ts — 分步生成逻辑

```typescript
// 品类分组顺序
const LAYER_ORDER = ['top', 'bottom', 'shoes'];

// 按品类分组服装
function groupByCategory(garments: Garment[]): Garment[][] {
  const groups = new Map<string, Garment[]>();
  for (const g of garments) {
    const key = g.category === 'dress' ? 'top' : g.category;
    if (!groups.has(key)) groups.set(key, []);
    groups.get(key)!.push(g);
  }
  return LAYER_ORDER.map(k => groups.get(k) || []).filter(g => g.length > 0);
}

// 分步执行
async function executeMultiStep(layers, bodyProfile, scene, quality) {
  let currentRefImages = [bodyProfile.frontPhotoUrl];
  const results = [];

  for (const layer of layers) {
    const stepRef = [...currentRefImages];
    for (const g of layer) {
      if (stepRef.length >= 4) break;
      stepRef.push(g.processedImage || g.originalImage);
    }
    const prompt = buildStepPrompt(bodyProfile, layer, scene, results.length === 0);
    const url = await callImageApi(prompt, stepRef, quality);
    results.push({ category: layer[0].category, url });
    currentRefImages = [url]; // 下一步以当前结果为参考
  }

  return results;
}
```

#### 2.3.2 Prompt 设计

每步 Prompt 不同：

**第一步（穿上衣）**：
```
请让图中人物穿上以下服装：
- 上衣：白色圆领T恤，纯棉，标准版型
保持面部特征、发型、体型完全一致。背景：专业摄影棚。
```

**第二步（加裤装）**：
```
请在现有穿搭基础上，让图中人物继续穿上以下服装：
- 裤装：深蓝色修身牛仔裤
注意保持已有上衣不变，新增裤装要自然搭配。保持面部特征一致。
```

**第三步（加鞋履）**：
```
请在现有穿搭基础上，让图中人物穿上以下鞋履：
- 鞋履：白色低帮运动鞋
注意保持已有服装不变，鞋履要自然穿着。保持面部特征一致。
```

### 2.4 兼容性

- 单件服装：走原有逻辑，不触发分步
- 2 件不同品类：2 步生成
- 3 件不同品类：3 步生成
- 同品类多件（如 2 件上衣）：合并为一步，只取第一个的 reference_images

```prisma
model Outfit {
  id          String   @id @default(uuid()) @db.Uuid
```

## 四、P1：AI 穿搭推荐

### 4.1 功能描述

用户不手动选服装，由 AI 根据身材数据 + 风格偏好 + 场景自动推荐整套穿搭方案。

### 4.2 技术方案

调用 mimo-v2.5 语言模型，输入用户的身材数据和风格偏好，输出推荐的服装组合。

#### Prompt 设计

```
你是一个专业穿搭顾问。根据以下用户信息推荐一套完整穿搭：

## 用户信息
- 性别：男
- 身高：175cm，体重：68kg
- 体型：标准型
- 风格偏好：休闲 70%，正式 30%

## 场景
通勤（办公室环境）

## 衣橱可用单品
1. 白色圆领纯棉T恤（休闲，标准版型）
2. 黑色休闲西装外套（商务，修身版型）
3. 深蓝色修身牛仔裤（休闲街头）
4. 卡其色休闲长裤（商务休闲）
5. 白色低帮运动鞋（百搭）
6. 灰色运动卫衣（运动休闲）

## 要求
- 推荐 1 套穿搭（上衣 + 裤装 + 鞋履）
- 考虑身材特点和场景需求
- 返回 JSON 格式：
{
  "outfit_name": "方案名称",
  "garment_ids": ["id1", "id2", "id3"],
  "reason": "推荐理由",
  "style_score": 8,
  "scene_match": "场景匹配度说明"
}
```

### 4.3 API 接口

## 六、P2：穿搭评分与一键换装

### 6.1 穿搭评分

调用 mimo-v2.5 对已生成的试穿效果图进行多维度评分：

```
请评估这套穿搭的整体效果（1-10分）：
1. 风格协调度：服装之间的颜色、风格是否搭配
2. 场景匹配度：穿搭是否适合指定场景
3. 体型适配度：版型是否适合用户身材
4. 整体观感：视觉效果和时尚感
返回 JSON：{"overall": 8, "style": 7, "scene": 9, "body": 8, "feedback": "..."}
```

### 6.2 一键换装

在已有穿搭方案中替换其中一件单品：

```
POST /api/outfits/:id/swap
{
  "replaceGarmentId": "uuid-of-old-shirt",
  "newGarmentId": "uuid-of-new-shirt"
}
```

替换后自动重新生成试穿效果图。

## 八、风险与注意事项

| 风险 | 应对 |
|------|------|
| 分步生成累积误差（每步面部偏移） | Prompt 中每步强调"保持面部一致"，必要时质量自检 |
| 多步生成耗时长（3步×15秒=45秒） | 前端展示步骤进度，给用户预期；提供"快速预览"低质量模式 |
| AI 推荐结果不符合用户衣橱 | Prompt 中传入实际可用单品列表，限制推荐范围 |
| reference_images 传递中间结果的 URL 有效期 | hfsyapi 返回的 URL 是临时的，需在有效期内完成下一步调用 |
