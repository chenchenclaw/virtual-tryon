# 虚拟试穿二期 — 版型与姿态估计技术方案

## 一、背景与目标

### 1.1 问题描述

一期方案中，服装版型（修身/宽松/廓形等）通过文字描述注入 Prompt，生成效果已有一定可信度。但存在以下不足：

1. **体态差异被忽略**：同样穿修身衬衫，含胸体态和挺胸体态的穿着效果完全不同，但一期无法区分
2. **版型感知粗糙**：AI 只知道"这是修身款"，但不知道修身款在用户具体身材上哪些部位会紧、哪些部位会松
3. **合身度不可见**：用户看不出衣服穿在身上哪里合身、哪里有余量，只能凭直觉判断

### 1.2 二期目标

引入 **姿态估计 + 体态分析**，实现：

- 自动从用户照片提取体态特征（含胸、溜肩、O 型腿等）
- 建立 "版型 × 体态" 矩阵，生成差异化的合身描述
- 在试穿效果图上可视化标注松紧区域
- 让试穿效果从"穿上了"提升到"穿得像"

### 2.2 姿态估计技术选型

| 方案 | 精度 | 速度 | 部署方式 | 推荐 |
|------|------|------|---------|------|
| **MediaPipe Pose** | 中高 | 极快（<50ms） | **浏览器端** | **首选** |
| DWPose | 高 | 中（~200ms） | 后端 GPU | 高精度备选 |
| OpenPose | 高 | 慢（~500ms） | 后端 GPU | 不推荐 |
| MoveNet | 中 | 快（~30ms） | 浏览器/后端 | 轻量备选 |

**推荐 MediaPipe Pose，理由**：

1. **浏览器端运行**：用户照片无需上传到后端做姿态估计，减少隐私风险和网络延迟
2. **33 个全身关键点**：覆盖肩、肘、腕、髋、膝、踝等所有需要的关节点
3. **WebAssembly 加速**：在主流浏览器上推理速度 < 50ms
4. **Google 维护**：长期稳定更新，社区活跃
5. **精度足够**：体态分析不需要亚像素精度，MediaPipe 完全满足需求

class PoseEstimator {

  async initialize() {
    const vision = await FilesetResolver.forVisionTasks(

        delegate: 'GPU'
      },
      runningMode: 'IMAGE',
      numPoses: 1
    });
  }

    const result = this.poseLandmarker.detect(imageElement);
    return this.extractKeyPoints(result);

    const landmarks = result.landmarks[0];
    return {
      leftShoulder: landmarks[11],
      rightShoulder: landmarks[12],
      leftHip: landmarks[23],
      rightHip: landmarks[24],
      leftKnee: landmarks[25],
      rightKnee: landmarks[26],
      leftAnkle: landmarks[27],
      rightAnkle: landmarks[28],
      nose: landmarks[0],
      // ... 其他关键点
    };
  }
}

## 三、体态特征分析算法

### 3.1 关键角度计算

从 MediaPipe 关键点计算体态角度：

```typescript
interface BodyAngles {
  shoulderTilt: number;      // 肩膀倾斜角（度），0=水平，正=左高右低
  spineDeviation: number;    // 脊柱偏离角（度），0=挺直，正=前弯（含胸）
  pelvicTilt: number;        // 骨盆倾斜角（度），0=水平
  leftKneeAngle: number;     // 左膝角度，180=伸直
  rightKneeAngle: number;    // 右膝角度，180=伸直
  torsoLean: number;         // 躯干前倾角（度），0=垂直
}

function calculateBodyAngles(pose: PoseResult): BodyAngles {
  // 肩膀倾斜角：左右肩连线与水平线的夹角
  const shoulderTilt = Math.atan2(
    pose.leftShoulder.y - pose.rightShoulder.y,
    pose.leftShoulder.x - pose.rightShoulder.x
  ) * (180 / Math.PI) - 90;

  // 脊柱弯曲角：肩中点-髋中点连线与垂直线的夹角
  const shoulderMid = midpoint(pose.leftShoulder, pose.rightShoulder);
  const hipMid = midpoint(pose.leftHip, pose.rightHip);
  const spineDeviation = Math.atan2(
    shoulderMid.x - hipMid.x,
    hipMid.y - shoulderMid.y
  ) * (180 / Math.PI);

  // 膝关节角度：大腿-小腿夹角
  const leftKneeAngle = calculateAngle(pose.leftHip, pose.leftKnee, pose.leftAnkle);
  const rightKneeAngle = calculateAngle(pose.rightHip, pose.rightKnee, pose.rightAnkle);

  // ... 其他角度计算

  return { shoulderTilt, spineDeviation, pelvicTilt, leftKneeAngle, rightKneeAngle, torsoLean };
}
```

### 3.2 体态分类

将角度数据映射为体态标签：

```typescript
interface BodyPosture {
  shoulderType: '溜肩' | '正常' | '平肩' | '耸肩';
  chestType: '含胸' | '正常' | '挺胸';
  pelvicType: '前倾' | '正常' | '后倾';
  legType: 'O型腿' | '正常' | 'X型腿';
  overallPosture: string;  // 综合体态描述
}

function classifyBodyPosture(angles: BodyAngles): BodyPosture {
  // 肩膀分类
  let shoulderType: BodyPosture['shoulderType'];
  if (Math.abs(angles.shoulderTilt) > 8) {
    shoulderType = angles.shoulderTilt > 0 ? '溜肩' : '耸肩';  // 简化判断
  } else {
    shoulderType = '正常';
  }

  // 胸部/脊柱分类
  let chestType: BodyPosture['chestType'];
  if (angles.spineDeviation > 10) chestType = '含胸';
  else if (angles.spineDeviation < -5) chestType = '挺胸';
  else chestType = '正常';

  // 腿型分类（通过膝关节角度和踝关节距离判断）
  let legType: BodyPosture['legType'];
  const ankleDistance = distance(pose.leftAnkle, pose.rightAnkle);
  const kneeDistance = distance(pose.leftKnee, pose.rightKnee);
  const ratio = ankleDistance / kneeDistance;
  if (ratio > 1.15) legType = 'O型腿';
  else if (ratio < 0.85) legType = 'X型腿';
  else legType = '正常';

  // 生成综合体态描述
  const overallPosture = generatePostureDescription(shoulderType, chestType, pelvicType, legType);

  return { shoulderType, chestType, pelvicType, legType, overallPosture };
}
```

### 4.3 Prompt 增强组装

```typescript
function buildFitAwarePrompt(
  basePrompt: string,
  posture: BodyPosture,
  fitType: string,
  garmentCategory: string
): string {
  const matrixKey = getMatrixKey(posture, fitType, garmentCategory);
  const fitEffect = FIT_POSTURE_MATRIX[fitType]?.[matrixKey];

  if (!fitEffect || fitEffect.tightZones.length === 0) {
    // 体态对版型无显著影响，使用基础 Prompt
    return basePrompt;
  }

  // 追加体态感知指令
  return `${basePrompt}

## 体态适配要求（重要）
用户体态特征：${posture.overallPosture}
服装版型：${fitType}

请特别注意以下细节：
${fitEffect.promptHint}

紧绷区域（需表现轻微面料张力）：${fitEffect.tightZones.join('、')}
余量区域（需表现自然垂坠或微松）：${fitEffect.looseZones.join('、')}

注意：这些细节要自然真实，不要过度夸张。`;
}
```

## 五、合身度可视化标注（二期）

### 5.1 标注方案

在生成的试穿效果图上叠加合身度热力标注，让用户直观看到服装各部位的松紧状态：

```
┌─────────────────────────────────────────┐
│                                         │
│         ┌─ 肩部：合身 ✓                  │
│        /│\                              │
│       / │ \   ┌─ 胸部：适中              │
│      /  │  \  │                          │
│         │      ┌─ 腰部：略紧 ⚠           │
│         │     │                          │
│         │      ┌─ 臀部：合身 ✓           │
│        / \    │                          │
│       /   \                             │
│      /     \  ┌─ 膝部：紧绷 ⚠           │
│                                         │
│  整体合身度：⭐⭐⭐⭐ (4/5)               │
│  推荐尺码：M                             │
└─────────────────────────────────────────┘
```

### 5.2 实现方式

1. **Canvas 叠加层**：在效果图上方覆盖透明 Canvas，绘制标注线和标签
2. **关键区域定位**：根据 MediaPipe 关键点坐标定位身体各部位在图片上的位置
3. **颜色编码**：合身=绿色，适中=黄色，紧绷=橙色，过松=蓝色

```typescript
interface FitAnnotation {
  zone: string;           // 'shoulder' | 'chest' | 'waist' | 'hip' | 'knee'
  label: string;          // '合身' | '适中' | '略紧' | '紧绷' | '宽松'
  position: { x: number; y: number };  // 标注位置（图片坐标）
  color: string;          // 颜色编码
  detail: string;         // "余量约4cm"
}

function renderFitAnnotations(
  canvas: HTMLCanvasElement,
  image: HTMLImageElement,
  annotations: FitAnnotation[],
  pose: PoseResult
) {
  const ctx = canvas.getContext('2d')!;
  ctx.drawImage(image, 0, 0);

  for (const ann of annotations) {
    // 根据关键点计算标注位置
    const pos = getAnnotationPosition(ann.zone, pose, image);

    // 绘制标注线
    ctx.strokeStyle = ann.color;
    ctx.lineWidth = 2;
    ctx.setLineDash([5, 3]);
    ctx.beginPath();
    ctx.moveTo(pos.x, pos.y);
    ctx.lineTo(pos.x + 60, pos.y - 30);
    ctx.stroke();

    // 绘制标签
    ctx.setLineDash([]);
    ctx.fillStyle = ann.color;
    ctx.font = '12px sans-serif';
    ctx.fillText(`${ann.label} ${ann.detail}`, pos.x + 65, pos.y - 35);
  }
}
```
