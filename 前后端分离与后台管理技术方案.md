# 虚拟试穿系统 — 前后端分离 + 后台管理技术方案

## 一、概述

本次改造涉及三大变更：
1. **前后端分离**：后端从 Next.js API Routes 迁移到 Python FastAPI
2. **后台管理系统**：开发独立的管理后台，管理用户、数据、系统配置
3. **存储策略优化**：针对百人规模，合理划分数据库/浏览器本地存储，兼顾隐私与体验

**分支**：`feat/backend-separation`

## 四、存储策略（百人规模）

### 4.1 数据分类与存储方案

百人规模下，数据量有限，优先选择简单可靠的方案。

| 数据类型 | 存储位置 | 理由 |
|---------|---------|------|
| **用户账号** | PostgreSQL | 核心数据，需服务端验证 |
| **体型档案** | PostgreSQL | 跨设备同步，AI 调用需服务端访问 |
| **服装元数据** | PostgreSQL | 跨设备，管理后台需访问 |
| **服装图片** | 服务器本地 uploads/ | 百人规模不需要 OSS，本地足够 |
| **尺码表** | PostgreSQL | 结构化数据，推荐算法需要 |
| **试穿任务/结果** | PostgreSQL | 历史记录，管理后台需监控 |
| **穿搭方案** | PostgreSQL | 跨设备，可分享 |
| **用户偏好/设置** | 浏览器 localStorage | 轻量，无需服务端 |
| **表单草稿** | 浏览器 localStorage | 临时数据，防刷新丢失 |
| **UI 状态** | 浏览器 localStorage | 主题、语言、布局偏好 |
| **搜索历史** | 浏览器 localStorage | 隐私敏感，不需要服务端 |
| **JWT Token** | 浏览器 httpOnly Cookie | 安全存储 |

### 4.2 存在浏览器本地的数据（localStorage）

```typescript
// 用户偏好（不涉及隐私）
const localKeys = {
  theme: 'light' | 'dark',                    // 主题
  language: 'zh' | 'en',                       // 语言
  tryonDefaultScene: 'studio',                 // 默认试穿场景
  tryonDefaultPose: 'front_standing',          // 默认姿势
  wardrobeSortBy: 'createdAt',                 // 衣橱排序方式
  bodyFormDraft: { height: 175, ... },         // 体型表单草稿
  recentSearches: ['白色T恤', '牛仔裤'],       // 搜索历史
};
```

**原则**：浏览器本地只存「体验偏好」类数据，不存业务核心数据。

### 4.3 隐私保护策略

#### 4.3.1 数据分级

| 级别 | 数据 | 处理方式 |
|------|------|---------|
| L1 极敏感 | 用户照片（正面/侧面） | 仅用于 AI 生成，不公开，可随时删除 |
| L2 敏感 | 身材数据（围度/体重） | 仅用户本人和 AI 服务可访问 |
| L3 一般 | 服装图片、穿搭方案 | 用户可控公开/私密 |
| L4 公开 | 昵称、风格偏好 | 可用于社区展示 |

#### 4.3.2 隐私措施

1. **用户照片**：
   - 存储在服务器本地，不上传到第三方存储
   - 传给 AI API 时仅发送 URL（hfsyapi/mimo 中转站），不存储在 AI 平台
   - 用户删除账号时，级联删除所有照片

2. **身材数据**：
   - 仅存储在 PostgreSQL，API 返回时脱敏（管理后台不展示具体围度数值）
   - 不用于广告推荐或用户画像

3. **数据删除**：
   - 支持用户自助删除账号及所有关联数据
   - 管理员删除用户时需二次确认

4. **API 调用隐私**：
   - 传给 hfsyapi 的 reference_images 是临时 URL，API 调用完成后不保留
   - Prompt 中的身材数据仅用于当次生成，不做持久化日志

#### 4.3.3 管理后台数据脱敏

管理后台查看用户数据时：
- 身材数据：仅展示「175cm / 68kg」，不展示具体围度
- 用户照片：仅展示缩略图，不提供原图下载
- 试穿结果：仅展示缩略图

## 五、数据库迁移

### 5.1 保持现有 Prisma Schema

数据库结构不变，FastAPI 通过 SQLAlchemy 映射同一张表。Alembic 管理迁移。

### 5.2 SQLAlchemy 模型映射

```python
# backend/app/models/user.py
from sqlalchemy import Column, String, Integer, DateTime, JSON
from app.core.database import Base

class User(Base):
    __tablename__ = "users"

    id = Column(String, primary_key=True)
    email = Column(String, unique=True)
    phone = Column(String, unique=True, nullable=True)
    password_hash = Column(String)
    nickname = Column(String)
    avatar_url = Column(String, nullable=True)
    gender = Column(String, nullable=True)
    birth_year = Column(Integer, nullable=True)
    style_preferences = Column(JSON, nullable=True)
    created_at = Column(DateTime)
    updated_at = Column(DateTime)
```

```sql
CREATE TABLE admins (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR NOT NULL,
    role VARCHAR(20) DEFAULT 'admin',  -- super_admin / admin / viewer
```

## 六、前端改造

### 6.1 Next.js 改为纯前端

去掉所有 `src/app/api/` 目录，API 调用统一通过 axios/fetch 调用 FastAPI。

#### API 客户端封装

```typescript
// frontend/src/lib/api.ts
import axios from 'axios';

const api = axios.create({
  baseURL: process.env.NEXT_PUBLIC_API_URL || 'http://localhost:8000/api/v1',
  withCredentials: true,  // 携带 httpOnly cookie
});

// 请求拦截：添加 token
api.interceptors.request.use((config) => {
  // token 由后端通过 httpOnly cookie 设置，浏览器自动携带
  return config;
});

// 响应拦截：401 跳转登录
api.interceptors.response.use(
  (res) => res,
  (err) => {
    if (err.response?.status === 401) {
      window.location.href = '/login';
    }
    return Promise.reject(err);
  }
);

export default api;
```

### 6.2 前端调用示例

```typescript
// 原来：fetch('/api/garments')
// 改为：api.get('/garments')

// 原来：fetch('/api/tryon', { method: 'POST', body: ... })
// 改为：api.post('/tryon', { garmentIds, scene, pose })
```

### 6.3 CORS 配置

FastAPI 需配置 CORS 允许前端域名：

```python
# backend/app/main.py
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000", "http://localhost:3001"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

## 九、风险与注意事项

| 风险 | 应对 |
|------|------|
| 迁移期间数据一致性 | 保持同一 PostgreSQL，Alembic 迁移脚本与 Prisma schema 同步 |
| 前后端联调成本 | FastAPI 自动生成 OpenAPI 文档，前端按文档对接 |
| Celery 增加运维复杂度 | Docker Compose 统一编排，开发环境可用同步模式替代 |
| 图片文件跨服务共享 | uploads/ 目录通过 Docker volume 共享 |
| 管理后台安全 | 独立认证体系，强密码 + 角色权限 |
